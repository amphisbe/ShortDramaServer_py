第一层：采集端详细设计

多形态入口覆盖不同场景。每个采集端内置预处理引擎，将富媒体统一转为结构化文字后推入中枢收件箱。

🖥
Chrome/Firefox 插件
网页剪藏、选中文字+截图一键发送、链接分享。适合电脑端浏览时快速采集。
浏览器端
💬
微信服务号/小程序
聊天文件转发、公众号文章、语音消息直接发送。移动端碎片采集的首选入口。
移动端
📂
桌面分享目标
macOS Share Menu 或 Windows "发送到"菜单。本地文件、截图一键送出。
桌面端

预处理引擎（端侧）

🎤 语音 → 文字
Whisper 本地模型或云端 STT，支持多语言
🎬 视频 → 文字简报
分离音频轨 → STT → 带时间戳文字稿
📷 OCR 图片识别
截图/照片文字提取，原图保留为附件
📄 PDF 文本提取
内嵌文本优先，扫描件 OCR 兜底

第二层：中枢调度中心

部署在公网服务器的团队 Agent 调度中心。核心原则：不拆解任务、不执行具体业务，只负责接收、理解、分发、跟踪。

边界约束：中枢不做详细内容编辑、不替代人做决策、不执行下游业务逻辑。它只回答一个问题——"这条内容该交给谁？"——然后定期询问"处理得怎么样了？"

核心能力

🧠 内容理解
AI 判断类型、主题、紧急度、关联项目
👤 人员路由
规则 + AI 推理，匹配最佳接收人
🔀 系统路由
决定进 Obsidian 还是 Notion，哪个文件夹/数据库
📊 状态跟踪
全生命周期可见，超时提醒，周期汇总

第三层：知识落库

📁
Obsidian Vault
深度学习区。AI 提炼内容自动生成 Markdown，建立双链知识图谱。

适合：个人思考、深度整理、长期洞察
个人区
📋
Notion Workspace
团队协作区。任务卡片、共享文档、看板、周报数据库。

适合：团队共享、任务跟踪、公开文档

端到端数据流

sequenceDiagram
    actor U as 👤 团队成员
    participant C as 📥 采集端
    participant H as 🧠 中枢调度
    participant AI as 🤖 AI 引擎
    participant PM as 👔 项目经理
    participant O as 📁 Obsidian
    participant N as 📋 Notion

    U->>C: 粘贴链接 / 分享截图 / 转发文件
    C->>C: 预处理（STT/OCR/去重/标准化）
    C->>H: 推送标准化内容

    H->>AI: 分析内容属性
    AI-->>H: 分类/标签/项目归属/紧急度/推荐接收人

    alt 深度参考类
        H->>O: 写入 Markdown + 建立双链
    else 协作事项类
        H->>N: 创建任务卡片 / 文档
        H->>PM: 通知「有新内容待处理」
    end

    loop 定期跟踪
        H->>PM: 询问处理进度
        PM-->>H: 反馈状态
        H->>H: 更新生命周期状态
    end

    Note over H: 超时未完结 → 自动催办通知
    Note over H: 周期末 → 生成汇总报告

端到端数据流：从采集到归档，中枢全程调度但不拆解执行
